# Reviewed SP11 kernel reconstruction

The experimental source is reconstructed in four incremental stages on Linux
stable `v7.1.3`: a conservative SP11/HID-over-SPI prerequisite, the separate
provenance-reviewed camera branch, a focused touch-driver autoload fix, and a
delayed post-resume tablet-mode resynchronization candidate.

## Exact-history method

```sh
git clone https://git.kernel.org/pub/scm/linux/kernel/git/stable/linux.git linux
cd linux
git checkout v7.1.3
git fetch /path/to/sp11-alpha/kernel/sp11-sanitized2.bundle \
  refs/heads/sp11-sanitized2:refs/heads/sp11-sanitized2
git switch sp11-sanitized2
git rev-parse HEAD^{commit} HEAD^{tree}
```

The final command must print:

```text
2ace98eb6ef18cbd48074eed9f5b585d19ce398b
3924711c8ab3ee5c0208d214a4433eafc271df42
```

The bundle is intentionally incremental. It contains the complete sanitized
delta but requires base commit
`199c9959d3a9b53f346c221757fc7ac507fbac50` from Linux `v7.1.3`.

## End-state patch method

For a compact review or rebase:

```sh
git checkout v7.1.3
git apply --check /path/to/sp11-alpha/kernel/sp11-sanitized2.patch
git apply /path/to/sp11-alpha/kernel/sp11-sanitized2.patch
```

This reproduces the final files but not the curated 12-commit history.

## Configuration and build

Merge `kernel/config` with `kernel/camera-review.config.fragment` in the
out-of-tree build directory. The maintained helper performs that merge and
verifies the exact camera source identity:

```sh
./scripts/build-kernel.sh \
  --source /path/to/linux \
  --output /path/to/build \
  --jobs "$(nproc)"
```

See `BUILDINFO` and `docs/BUILD.md` for build and verification details.
`Module.symvers` is generated by a clean build and documents the matching
module ABI. Review8 deliberately keeps the byte-identical review4 merged
configuration because the review5 through review8 corrections change source
only. The build helper's explicit `KERNELRELEASE=7.1.3-sp11-camera-review8` overrides
the retained review4 `CONFIG_LOCALVERSION` and defines the authoritative
release identity.
It also runs the declared Python 3.14.6 with `-S`, excluding site packages from
Kconfig probes so the merged configuration remains reproducible.

## Experimental camera branch

`sp11-camera-review.bundle` adds 13 reviewed commits on top of the exact
`sp11-sanitized2` tip. It is deliberately separate from the camera-free
candidate, does not contain the withdrawn Practical8 history, and has passed
bounded Wi-Fi, Bluetooth, three-camera, and IR-illuminator hardware tests on
the target Surface.

```sh
git fetch /path/to/sp11-alpha/kernel/sp11-camera-review.bundle \
  refs/heads/sp11-camera-review:refs/heads/sp11-camera-review
git switch sp11-camera-review
git rev-parse HEAD^{commit} HEAD^{tree}
```

The final command must print:

```text
675d89b381d8b730a3f2eff1086875481ee5b515
03c278405e6d2fd0ffa1fd4cad860ef45c7adbbc
```

For patch review, apply `sp11-camera-review.patch` to the
`sp11-sanitized2` tip. Enable the camera-specific options from
`camera-review.config.fragment`. See `docs/CAMERA-REVIEW.md` for provenance,
validation, and remaining test gates.

## Touch SPI autoload correction

`sp11-touch-spi-autoload.bundle` adds one commit on top of the exact camera
tip. Review4 exposed the touchscreen as `spi:g6-touch-digitizer`, but its
`spi_hid_of` module advertised only `spi:microsoft,g6-touch-digitizer`; udev
therefore could not autoload the driver. Explicitly loading the module restored
the physical touchscreen/stylus and both iptsd virtual devices. The correction
restores the SPI ID used by the actual modalias without changing the DTB.

```sh
git fetch /path/to/sp11-alpha/kernel/sp11-touch-spi-autoload.bundle \
  refs/heads/fix/touch-spi-autoload:refs/heads/fix/touch-spi-autoload
git switch fix/touch-spi-autoload
git rev-parse HEAD^{commit} HEAD^{tree}
```

The final command must print:

```text
86fc94c58a89a56c7ceb57b42c6025b2569da56d
4624d85595964242c26d7042106d068cbbdd9977
```

For patch review, apply `sp11-touch-spi-autoload.patch` to the exact
`sp11-camera-review` tip.

## Tablet-mode resume resynchronization candidate

`sp11-tablet-mode-resume-resync.bundle` adds three commits on top of the exact
touch-autoload tip. The Surface Aggregator tablet-mode switch queries the
controller immediately on resume, then queries it once more after two seconds.
It also uses the KIP connection event to query every two seconds for a bounded
30-second settling window when the Flex Keyboard is attached or detached.
Transient values outside the valid posture range are not cached, and the
sequence stops on the first valid controller response. Each path reports a new
input state only when the controller itself says that the posture changed.

```sh
git fetch /path/to/sp11-alpha/kernel/sp11-tablet-mode-resume-resync.bundle \
  refs/heads/fix/tablet-mode-resume-resync:refs/heads/fix/tablet-mode-resume-resync
git switch fix/tablet-mode-resume-resync
git rev-parse HEAD^{commit} HEAD^{tree}
```

The final command must print:

```text
940bbc856a120e6f967f9dbaf825d5473bfae664
62edee5183ed3b42ee3a2f9f0c71066c3ab87742
```

For patch review, apply the three-message
`sp11-tablet-mode-resume-resync.patch` series to the exact touch-autoload tip.
Review6 exposed the missed reattach-notification case; review7 then exposed a
transient raw-zero controller response. Review8's live module passed repeated
target-hardware reattachment, while exact full-kernel qualification remains
pending.
